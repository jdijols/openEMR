# AgentForge V1 — Clinical Co-Pilot Implementation Task List

This document is the **execution map** for shipping the V1 Clinical Co-Pilot by Sun May 3, 12:00 local. It is rooted in [`PRD.md`](../../../PRD.md) and ordered by **ship gates** (dependency-first), **not** by PRD section order. Every task points back to its PRD section so anything here can be re-derived from the spec.

## How to read this file

- **Gates run mostly in order, with one explicit parallel lane after Gate 1.** Do not start Gate N+1 until Gate N's exit criteria are green, with the documented exception below.
- **Tests are not symmetric.** Security and agent-correctness tasks are **test-first** (write the unit/security test before or with the code). UI rendering, scaffolding, and infra are **smoke + manual demo**, not test-first.
- **Cut tier** = which [`PRD.md` §15.1](../../../PRD.md#151-cuttable-scope-tiers-apply-in-order-if-you-must-cut) tier this task can be dropped to recover schedule. `MUST` means the task cannot be cut without violating §1.5 anti-success criteria.
- **Done proof** is the literal artifact, command output, or scenario name a reviewer can re-check. Vague proofs are bugs in this file.
- **Owner** column is intentionally omitted — single engineer + AI pair.

## Stop-the-line tests (non-negotiable)

If any of these fail at any point in the schedule, stop feature work and fix before continuing. They map directly to [`PRD.md` §1.5](../../../PRD.md#15-anti-success-criteria-what-would-break-the-demo) and [§8](../../../PRD.md#8-security-baseline).

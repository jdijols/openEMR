## How to read this file

- **Gates run in dependency order.** Within a gate, sub-tasks may run in parallel where the **Depends on** column allows.
- **Tests are not symmetric.** Security and agent-correctness tasks are **test-first** (write the unit/integration test before or with the code). UI rendering, scaffolding, and infra are **smoke + manual demo**, not test-first.
- **Cut tier** = which scope tier this task can be dropped to recover schedule. `MUST` means the task cannot be cut without violating §1 anti-success criteria of [`W2_ARCHITECTURE.md`](W2_ARCHITECTURE.md). Lower tiers cut earlier (tier 1 first, tier 6 last).
- **Done proof** is the literal artifact, command output, or scenario name a reviewer can re-check. Vague proofs are bugs in this file.
- **Owner** column intentionally omitted — single engineer + AI pair.
- **Status markers:** `[ ]` open · `[~]` in progress · `[x]` done with proof · `[-]` cut per a tier (note tier in line).
